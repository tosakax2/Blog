Hexo とは何か？ブログを作るための第一歩

Hexo とは？

Hexo は「静的サイトジェネレーター」と呼ばれるツールの一つです。静的サイトジェネレーターは、コードを使わずにブログやウェブサイトを簡単に作成できる仕組みを提供します。「静的サイト」とは、ページの内容がサーバー側で固定されており、訪問者がページを読み込むたびに内容が再生成される「動的サイト」と異なります。そのため、静的サイトは高速で、セキュリティリスクが低いのが特徴です。

Hexo の魅力を簡単にまとめると以下の通りです：

  • シンプルで初心者にも使いやすい：一度環境を整えれば、簡単なコマンド操作でブログが作成できます。
  • 無料で運用可能：ブログ作成から公開までコストゼロで行えます。
  • テーマやプラグインが豊富：カスタマイズ性が高く、デザインも自由自在。

これらの特徴から、Hexo は「非技術者でもプロフェッショナルなブログを作れるツール」として広く愛用されています。

必要な準備

Hexo を始める前に、以下を準備しましょう：

  1. パソコン
    • Windows、Mac、Linux のどれでも OK です。Hexo はほとんどの OS で動作します。
  2. インターネット接続
    • 必要なソフトウェアをインターネットからダウンロードするために必要です。
  3. 基本的なパソコン操作スキル
    • ファイルやフォルダの作成・削除ができる程度で大丈夫です。また、簡単な文字入力も求められます。
    • ※Hexo ではコマンドプロンプトやターミナルという画面を使いますが、詳しく解説するので安心してください。

Hexo のインストールと初期設定

必要なツールのインストール

Hexo を動かすためには、以下のツールをインストールする必要があります。全て無料で利用できますのでご安心ください。

Node.js のインストール

Node.js は、JavaScript を動かすためのツールです。Hexo はこの Node.js の上で動作します。以下の手順でインストールしましょう。

  1. Node.js の公式サイトにアクセス
    • Node.js 公式サイト に移動します。
  2. LTS バージョンを選択してダウンロード
    • 「LTS（長期サポート）」と書かれたバージョンをダウンロードしてください。LTS は安定性が高く、初心者にも推奨されます。
  3. インストーラーを実行
    • ダウンロードしたファイルをダブルクリックしてインストールを開始します。画面の指示に従うだけで OK です。
  4. 動作確認

    • インストール後、以下の手順で正常にインストールされているか確認します。

      • Windows の場合：「コマンドプロンプト」を開きます。Mac/Linux の場合：「ターミナル」を開きます。

      • 次のコマンドを入力して Enter を押してください：正常にインストールされている場合、v18.** などのバージョン番号が表示されます。

        node -v

npm とは何か？

Node.js をインストールすると、自動的に「npm（エヌピーエム）」というツールも使えるようになります。npm は「Node.js のパッケージ管理ツール」で、Hexo をインストールしたり、必要な追加機能を簡単に設定するために使います。

動作確認は次のコマンドを実行してください：

npm -v

バージョン番号が表示されれば OK です。

Hexo のインストール

必要なツールが揃ったら、いよいよ Hexo 本体をインストールします。以下の手順に従って進めてください。

  1. グローバルインストール

    コマンドプロンプトまたはターミナルで以下を入力して Hexo をインストールします：

    npm install -g hexo-cli

    ※「-g」は「グローバルインストール」を意味します。この設定により、どのフォルダからでも Hexo を使えるようになります。

  2. インストール確認

    Hexo が正しくインストールされたか確認するには、以下のコマンドを入力してください：

    hexo -v

    Hexo と Node.js、npm のバージョンが表示されれば成功です。

プロジェクトの作成

Hexo でブログを作るには、プロジェクトという単位で作業を始めます。以下の手順を実行してください。

  1. 新しいフォルダを作成

    Hexo 用のフォルダを作りましょう。例えば、「my-blog」という名前のフォルダを作成します。次に、このフォルダに移動します：

    mkdir my-blog
cd my-blog

  2. Hexo の初期化

    フォルダ内で以下のコマンドを入力してプロジェクトを初期化します：

    hexo init

    この操作で、必要なファイルやフォルダが自動的に作成されます。

  3. 依存パッケージのインストール

    Hexo プロジェクトを動かすために、依存パッケージをインストールします：

    npm install

テーマの選び方と適用

Hexo ではテーマ（デザインテンプレート）を簡単に適用できます。まずは、Hexo が標準で用意している「landscape」というテーマを使ってみましょう。

  1. テーマの確認

    初期化したフォルダ内の_config.ymlという設定ファイルを開きます。themeという項目が「landscape」になっていることを確認してください。

  2. テーマを変更したい場合

    他のテーマを使いたい場合は、Hexo の公式サイト（Hexo テーマ一覧）から好みのデザインを選び、インストール方法に従って適用します。

記事の作成とローカル環境での確認

記事作成の基本手順

Hexo では、記事を作成する際に「Markdown（マークダウン）」という記述方式を使います。Markdown は、文章に簡単な記号を追加するだけで、見出しやリスト、リンクなどを表現できる便利な記法です。

新しい記事を作成するコマンド

記事を作成するには、次のコマンドを使います。

hexo new post "記事のタイトル"

例えば、「初めての記事」というタイトルの記事を作成したい場合：

hexo new post "初めての記事"

この操作により、source/_posts フォルダ内に「初めての記事.md」という Markdown ファイルが作成されます。

Markdown の基本記法

作成されたファイルをテキストエディタ（VS Code やメモ帳など）で開き、内容を書き換えてみましょう。以下は Markdown の簡単な記法例です：

  • 見出し

    # 見出し 1

## 見出し 2

### 見出し 3

    見出しのレベルに応じて#を増やします。

  • 太字

    **太字のテキスト**

  • 箇条書きリスト

    - 項目 1
- 項目 2
- 項目 3

  • リンク

    [リンクのテキスト](https://example.com)

試しに次のような内容を記述してみてください：

---

title: 初めての記事
date: 2024-12-11 10:00:00
tags:

- サンプル
- ブログ

# 初めての記事へようこそ

これは Hexo で作成した最初のブログ記事です！

## Markdown の練習

- これは箇条書きのサンプルです。
- **太字**や*斜体*も使えます。

[こちらをクリックするとリンクに飛びます](https://hexo.io)

ローカル環境でのプレビュー

記事を書いたら、公開する前にローカル環境で確認できます。

サーバーを起動する

次のコマンドを実行して Hexo のローカルサーバーを起動します：

hexo server

コマンドを実行すると、次のような表示が出るはずです：

INFO  Hexo is running at http://localhost:4000

この URL（http://localhost:4000）をブラウザで開くと、自分のブログがプレビューできます。

記事の確認

ブラウザでプレビューしたら、新しく作成した記事がブログに反映されているか確認してみましょう。記事のタイトルや内容が正しく表示されていれば成功です！

サーバーを停止する

ローカルサーバーを停止するには、コマンドプロンプトまたはターミナルの画面で「Ctrl + C」を押します。

Cloudflare Pages でのブログ公開

Hexo でブログを作成したら、いよいよインターネット上に公開します。ここでは、Cloudflare Pagesという無料で簡単に使えるサービスを使って公開する手順を説明します。

Cloudflare Pages とは？

Cloudflare Pages は、静的サイトを簡単に公開できるホスティングサービスです。以下の特徴があります：

  • 無料で利用可能：ページ数やトラフィック量の制限がなく、個人用途に最適。
  • 簡単な設定：GitHub との連携で自動的にブログを公開可能。
  • 高速で安全：Cloudflare のインフラを利用するため、表示速度が速くセキュアです。

GitHub アカウントの作成とリポジトリの準備

Cloudflare Pages は GitHub と連携して動作するため、まず GitHub アカウントとリポジトリを準備しましょう。

GitHub アカウントを作成する

  1. GitHub 公式サイト にアクセスします。
  2. 「Sign up（サインアップ）」をクリックして、アカウントを作成します。
    • ユーザー名、メールアドレス、パスワードを入力。
    • 指示に従って、認証を完了してください。

新しいリポジトリを作成する

リポジトリとは、GitHub 上でファイルを管理するフォルダのようなものです。次の手順でリポジトリを作成します：

  1. GitHub のホーム画面で「Repositories」タブをクリック。
  2. 「New」ボタンを押します。
  3. 以下を入力：
    • Repository name: 自分のブログの名前（例: my-blog）
    • Visibility: 「Public」を選択（無料プランの場合、Private は制限あり）
  4. 「Create repository」をクリックしてリポジトリを作成します。

Hexo のブログを GitHub にアップロード

作成したブログを GitHub にアップロードするために、以下の手順を実行します。

Git をインストールする

Git はファイルを GitHub に送るためのツールです。以下のリンクからダウンロードしてインストールしてください：

  • Git 公式サイト

インストール後、以下のコマンドで正しくインストールされたか確認します：

git --version

バージョン番号が表示されれば成功です。

Hexo ブログを Git で初期化

  1. Hexo のフォルダに移動します（例: cd my-blog）。

  2. 次のコマンドを入力して Git を初期化します：

    git init

  3. 作成したリポジトリをリンクします：

    git remote add origin https://github.com/あなたのGitHubユーザー名/my-blog.git

ファイルをアップロード

以下のコマンドを順番に実行して、Hexo のファイルを GitHub にアップロードします：

git add .
git commit -m "Initial commit"
git branch -M main
git push -u origin main

※「main」は現在のデフォルトブランチ名です。

GitHub のリポジトリページを開き、アップロードしたファイルが表示されていれば成功です！

Cloudflare Pages での公開

GitHub にファイルをアップロードしたら、Cloudflare Pages を使ってサイトを公開します。

Cloudflare アカウントの作成

  1. Cloudflare 公式サイト にアクセス。
  2. 「Sign Up（サインアップ）」をクリックしてアカウントを作成します。
    • 必要な情報（メールアドレス、パスワード）を入力します。

Cloudflare Pages のプロジェクト作成

  1. Cloudflare ダッシュボードにログインし、「Pages」を選択します。
  2. 「Create a project」をクリック。
  3. 「Connect to Git」ボタンを押し、GitHub アカウントを Cloudflare と連携します。
  4. 先ほど作成したリポジトリ（例: my-blog）を選択します。

ビルド設定

Cloudflare Pages では、Hexo で生成された静的ファイルを正しく公開するための設定を行います：

  1. Build command（ビルドコマンド）:

    npm run build

  2. Build output directory（出力先ディレクトリ）:

    public

  3. 必要な環境変数（Node.js のバージョンなど）があれば設定しますが、デフォルトで問題ない場合がほとんどです。

公開 URL を確認

設定を完了すると、Cloudflare Pages が自動的にビルドを実行します。数分後、公開用の URL（例: https://your-blog.pages.dev）が発行されます。この URL をブラウザで開き、ブログが表示されていれば成功です！

まとめ

Hexo と Cloudflare Pages を使うことで、ブログを簡単かつ無料で公開できるだけでなく、静的サイトの魅力を学び、他の人に技術を広める楽しさも得られます。これをきっかけに、より多くの人が手軽にウェブの世界に飛び込めるよう、ぜひ積極的に発信を続けてみてください！
